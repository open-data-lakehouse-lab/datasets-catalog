# Dataset Selection Process

This document outlines the criteria for selecting datasets for the Open Data Lakehouse Lab project.

## Selection Criteria

To ensure the quality, reliability, and openness of the datasets in our lakehouse, we evaluate candidates based on the following criteria:

* **Public availability**: The dataset must be publicly accessible without restrictive proprietary barriers.
* **Clear license**: The dataset must have a clear, open-source-compatible license.
* **API or download accessibility**: Preference is given to datasets accessible via standard APIs or stable download links.
* **Stable provider**: The data provider should be a reputable public or official organization.
* **Historical data availability**: Availability of historical records is crucial for analytics and trend analysis.
* **Refresh frequency**: The update frequency should be consistent and documented.
* **Schema clarity**: The data structure should be well-documented or easy to infer.
* **Weather MVP usefulness**: For the initial phase, the dataset must provide relevant weather information (temperature, precipitation, etc.).
* **Ease of ingestion**: Technical feasibility of ingesting the data into the lakehouse.
* **Reproducibility**: The data collection and ingestion process must be reproducible.
* **No private data**: Datasets must not contain any personally identifiable information (PII) or private data.
